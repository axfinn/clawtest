```
┌──────────────────────────────────────────────────────────┐
│                     API 路由层 (Routes)                  │
│         /api/v1/* - RESTful 接口                          │

│            业务逻辑处理、数据验证、事务管理                  │

│                   数据访问层 (Models)                      │

├──────────────────────────────────────────────────────────┤
│                      数据库层 (Database)                  │

```

### 设计原则

1. **单一职责**: 各层职责清晰，互不干扰
2. **依赖注入**: 降低模块间耦合
3. **RESTful**: 统一的 API 设计风格
4. **配置分离**: 环境配置与代码分离

## 5. 安全设计

### 5.1 认证

- JWT Token 认证
- Token 有效期: 访问令牌 24h, 刷新令牌 7天
- 密码加密存储 (SHA256 + salt)

### 5.2 权限控制

- 基于角色的权限控制 (RBAC)
- 接口级别权限验证

### 5.3 安全措施

- SQL 注入防护 (参数化查询)
- XSS 防护 (输入过滤)
- CSRF 防护
- 请求频率限制
